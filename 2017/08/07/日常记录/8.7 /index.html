坚持&改进&反省

才写了几天，就有点不想写了，足以见得坚持是一件多么难的事情。

”你想要得到的，最好的办法就是让自己配得上。“——查理芒格

今年以来，自我更新的频率明显变慢了，上半年找到的借口是“在创业公司，一周六天班”，换完工作，似乎又找到了新的借口”这儿的工作比创业公司占用的时间还要多“。

  • 不要寻找借口，不要抱怨，不要再说”没有时间“
  • 要保持学习，保持进步，读好书，思考改进。

工作上

总是在说新来这地方，想法得不到认可采纳，可实际上，你做的还不够。

  • 性能优化的具体实践上，你并没有找到关键问题点，虽然总在超额完成被分配的任务，但做的还不够好，没有解决问题，永远都是没有解决问题

  • 脚本，虽然上上周末做到80%，但不是100%永远也不是100%；

    况且，你有没有想过，需要你把这件事情做到100%么？这样把其它同事的事情做到他做不到的程度，真的利于一个team的团结向上么？

  • 工作报告、分享上，有落入形式主义陷阱的趋势，需要悬崖勒马

    重要的永远是内容，画龙点睛之笔应放在最后的最后

学习上

多久没看完过一本技术书了？多久没好好学英语了？多久没有把年初的计划拿来执行了？

——这样看来，离心智成熟还差的有一段距离啊

code & project

  • 今天凌晨3点或是4点，终于开通了Google Play的开发者账号（接下来，个人项目落地的事情，要按部就班得进行了：）

    这一天迟到了得有两年了，执行力还有待提高

    • 一是上架 SuperProfile，这是明天的闲时任务
    • 二是自学AI等UI工具，组建team离的还很远，简单的先自己来，这一步，预计需要持续1~3个月的时间
    • 三是修改MultiORG组织架构，以及目标调整，预计在8月底完成v0.1上线，伴随SuperProfile 1~2个版本的迭代

  • 也是在凌晨，开通了Github private repository 7刀/月的价格不低，但相信这一定会物超所值

题外话

不作死就不会死

今天回家路上瞎蹦跶，摔了一跤，人是有点得意忘形了么？我看是有点了
